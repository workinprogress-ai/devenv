# Issue Integration — Pairing Notes

Use the shared [Issue Artifact Integration](../../common/references/issue-artifact-integration.md) reference for the actual CLI flows and command patterns.

Pair-programming specific usage notes:

- During pairing, the local plan file remains the session source of truth while a phase is in progress.
- When the plan came from a GitHub issue artifact comment, republish only to the same `doc_id`.
- Do not auto-publish progress mid-discussion; wait for the explicit confirmation points defined in the pair-programming skill.
- For freeform discoveries, status notes, and adjacent bug discussion, use normal `issue-comment` / `issue-create` confirmation flow from the shared reference.
